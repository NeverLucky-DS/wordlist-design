## Почему его нельзя терять

Эти карточки **невоспроизводимы дёшево**. За ними стоят месяцы прогона флота
воркеров и реально потраченные токены на 10+ аккаунтах. Если файл пропадёт,
восстановить его можно только повторив весь прогон — это недели и деньги.

Всё остальное в проекте производное и пересобирается:

| Что | Восстановимо? |
|---|---|
| `enrichment.db` (карточки) | ❌ **только повторным прогоном Mistral** |
| `vocab.db` (словари-источники) | ⚠️ только если сохранены исходные дампы |
| Postgres `vocab_cards` (зеркало) | ✅ пересобирается из `enrichment.db` за ~15 с |
| Парадигмы из Wiktionary | ✅ импортируются из дампа за 75 с |
| `form_kind` / `form_of` | ✅ считаются из `vocab.db` за 4 с |

## Как восстановиться из него

```bash

   backend/app/vocab/vocab_data/enrichment.db
```

## Правила

1. **Не редактировать.** Не открывать на запись, не запускать по нему миграции,
   не натравливать `import_morphology.py` — у скрипта есть `--enrich-db`, гонять
   всегда на отдельной копии.
2. **Не удалять.** Даже если кажется, что место кончилось.
3. **Не коммитить.** 126 МБ бинарника в git — правило в `.gitignore` уже стоит
   (`enrichment-GOLDEN-*.db`). Этот `.md` наоборот отслеживается: он объясняет,
   почему рядом лежит неотслеживаемый файл, которого не должно быть жалко удалить,
   но которого удалять нельзя.
4. **Держать копию вне этой машины.** Один файл на одном диске — это не бэкап.
   Внешний диск или облако.

> ⚠️ `.gitignore` защищает от коммита, но **не** от `git clean -xfd` — эта
> команда сносит именно неотслеживаемые файлы, то есть в первую очередь этот.
> Отсюда пункт 4.
